# 执行计划目录

> 存放待执行的内容生成任务

---

## 什么是执行计划？

执行计划（Execution Plan）是 Harness Engineering 的核心工作单元。

每个执行计划描述一个具体的任务：
- 要补充什么内容
- 覆盖哪些知识点
- 预期输出什么文档

---

## 执行计划格式

```markdown
# 任务名称

## 任务目标
一句话描述要完成的任务

## 输入
- 依赖的前置知识
- 参考文档

## 输出
- 要创建的文档路径
- 预期包含的面试题数量

## 知识点覆盖
| 知识点 | 难度 | 考察重点 |
|--------|------|----------|
| xxx | 中级 | xxx |

## 参考资源
- 链接 1
- 链接 2

## 验收标准
- [ ] 包含至少 3 道面试题
- [ ] 有 Java 代码示例
- [ ] 有 mermaid 图表
- [ ] 符合 quality-standards
```

## 如何创建执行计划

1. 复制 `template.md` 为新的执行计划文件
2. 填写任务详情
3. 提交到本目录
4. 通知智能体（本汪）开始执行

---

## 任务状态流转

```
待认领 → 进行中 → 待审核 → 已完成
   ↑        ↓
   └──── 阻塞/放弃
```
