# Plan-and-Solve 范式

## 概念

Plan-and-Solve (计划-执行) 范式强调先制定完整计划，再按步骤执行。

## 执行流程

```
Plan → Step 1 → Step 2 → Step 3 → ... → Final Answer
```

## 特点

- **一次性规划**: 减少重复推理开销
- **全局视角**: 提前考虑所有步骤
- **计划可调整**: 支持动态修订计划

## 适用场景


- 任务目标明确的场景
- 步骤之间依赖关系清晰的场景
- 需要控制成本的场景（减少 LLM 调用次数）

## 与 ReAct 对比

| 维度 | Plan-and-Solve | ReAct |
|------|----------------|-------|
| 规划方式 | 一次性 | 逐步 |
| 灵活性 | 较低 | 较高 |
| 成本 | 较低 | 较高 |
| 适用场景 | 明确任务 | 探索性任务 |

## 面试题

1. **什么时候应该选择 Plan-and-Solve 而不是 ReAct？**
   - 任务步骤明确且稳定
   - 需要控制 API 调用成本
   - 不需要根据中间结果动态调整

2. **Plan-and-Solve 如何处理执行过程中的错误？**
   - 可以引入检查点和重试机制
   - 结合 Reflexion 范式进行错误恢复
