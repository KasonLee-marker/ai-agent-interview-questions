# Multi-Agent 范式

## 概念

Multi-Agent 范式通过多个 Agent 分工协作来完成复杂任务，模拟团队工作模式。

## 典型架构

```
┌─────────────┐
│ Orchestrator│  (协调器)
└──────┬──────┘
       │
   ┌───┴───┐
   ▼       ▼
┌──────┐ ┌──────┐
│AgentA│ │AgentB│  (执行者)
└──┬───┘ └──┬───┘
   │        │
   ▼        ▼
┌──────┐ ┌──────┐
│AgentC│ │AgentD│  (验证者/专家)
└──────┘ └──────┘
```

## 常见角色分工

| 角色 | 职责 |
|------|------|
| Planner | 制定执行计划 |
| Executor | 执行具体任务 |
| Reviewer | 检查结果质量 |
| Critic | 提出改进建议 |
| Researcher | 收集信息 |

## 协作模式

1. **Sequential**: 顺序执行，A → B → C
2. **Parallel**: 并行执行，A 和 B 同时进行
3. **Hierarchical**: 层级结构，Manager 分配任务
4. **Debate**: 多 Agent 辩论，达成共识

## 面试题

1. **Multi-Agent 系统如何解决冲突？**
   - 引入仲裁者角色
   - 投票机制
   - 置信度加权

2. **如何设计 Agent 之间的通信协议？**
   - 消息队列
   - 共享状态
   - 黑板系统

3. **Multi-Agent 和单体 Agent 相比有什么优劣？**
   - 优势：专业化、可扩展、容错性
   - 劣势：通信开销、协调复杂、调试困难
