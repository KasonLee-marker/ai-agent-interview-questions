# Reflexion 范式

## 概念

Reflexion 是一种让 Agent 能够自我评估和从错误中学习的范式。

## 执行流程

```
Act → Evaluate → Reflect → (Retry if needed) → Final Answer
```

## 核心组件

1. **Actor**: 执行任务的主体
2. **Evaluator**: 评估执行结果
3. **Self-Reflection**: 生成改进建议
4. **Memory**: 存储历史反思

## 特点

- **自我学习**: 从失败中积累经验
- **持续改进**: 每次错误都转化为知识
- **记忆机制**: 长期积累避免重复犯错

## 适用场景

- 容易出错的复杂任务
- 需要长期学习的场景
- 对准确性要求高的任务

## 面试题

1. **Reflexion 的记忆机制如何设计？**
   - 短期记忆：当前任务的上下文
   - 长期记忆：跨任务的反思总结
   - 向量数据库：相似任务的检索

2. **如何避免 Reflexion 陷入无限循环？**
   - 设置最大重试次数
   - 引入人工审核机制
   - 评估改进效果，无进展时终止

3. **Reflexion 和强化学习有什么区别？**
   - Reflexion 基于语言反馈而非奖励信号
   - 不需要训练，零样本学习
